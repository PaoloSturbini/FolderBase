What FolderBase is

FolderBase shows the real folders and files on your Mac in a table you can extend with custom columns — notes, numbers, dates, Kanban states, tags and links — without ever moving, copying or altering the original files. Large folders appear progressively and external changes are shown immediately while caches and the database reconcile in the background.

Metadata lives in a separate local database and follows your files even when you rename or move them. The goal is to bring a Notion/Airtable-like experience to real filesystem files: Finder only shows name, size and date; FolderBase lets you annotate, classify and organize the same content with your own fields.

Adding and opening a folder

  1. Open Settings, go to Folders and use Add folder. Each folder appears as a root; the up/down buttons persistently reorder it and – removes it.
  2. Roots are visually separated and scroll together in the sidebar. On launch the app automatically reopens the last folder you used.

Navigating files

  • Single click: selects a row.
  • Double click (or Return) on a subfolder name: enters it, just like Finder.
  • At the top you'll find Back / Forward / Up (↑) buttons and the current path.
  • The sidebar shows all added folder trees in your chosen order. The blue highlight follows the exact selected folder or subfolder.
  • You can drag files or folders from the table onto a subfolder row in the same table to move them into it. Hold Option to copy. You can also drop them onto a folder in the tree on the left.
Auto-refresh. The table updates itself when you add, remove or rename files from elsewhere (Finder, terminal…).

Columns and metadata

Every table shows Name, Type, Created and Size by default. The + Column button adds metadata columns whose configuration is inherited by every subfolder; parent-folder definitions win conflicts.

Editing metadata

Type directly into the cells: saving is automatic. Text notes are saved with a short delay so the app doesn't write to disk on every keystroke.

Managing columns

  • Resize / reorder: drag borders and headers.
  • Show / hide and Rename / Delete: from the Columns menu in the toolbar.
  • Width, order and visibility are persistent across sessions.

Sort, search, filter

  • Click a column header to sort; click again to reverse. Empty values always go to the bottom. Default order restores folders first, then by name.
  • The search bar filters by name and by any metadata value.
  • The Filters menu shows only items with certain Select/Kanban values; active filters appear as removable chips.

Multiple selection and bulk edit

Select several rows with Cmd-click / Shift-click. You'll see:

  • Edit — set the same metadata value across the whole selection, in a single operation.
  • Trash — move the items to the Trash.

Kanban view (board)

If the folder has a Kanban column, the table / board toggle at the top shows cards grouped by state. Drag a card to another column to change its state.

File operations

Right-click an item to: Open, Quick Look, Reveal in Finder, Rename, Move, Set metadata, Move to Trash.

  • From Settings you can create empty files and folders in the current directory.
  • Export CSV saves the current table honoring the active filters and sorting.

Templates

A template is a reusable set of columns. The button remains available in the root and every subfolder. Propagation preserves values and Select/Kanban options in compatible fields, while incompatible local structures remain independent.

Settings ⚙︎

  • Folders — recent folders and file/folder creation.
  • Appearance — automatic / light / dark theme and font size.
  • Language — Italian or English, applied immediately.
  • Templates — manage reusable sets of columns.
  • AI features — indexing, search/chat engines and Excluded AI sources. You can exclude individual files or folders and scan managed roots for hidden or generated-directory suggestions; suggestions are never applied automatically.
  • Maintenance — metadata realignment and cleanup.
  • Backup — complete manual or scheduled copies and restore.

Backup and restore

Each new backup is a single .sqlite file containing the metadata database, managed folders, templates and preferences, including AI settings and exclusions. You can create one immediately or schedule automatic copies with a configurable interval and retention limit.

Before restoring, FolderBase checks file integrity and creates a complete safety copy of the current state. Older database-only backups remain compatible. The rebuildable AI index is not copied; API keys remain protected in Keychain.

Where your data lives

Operational metadata and the derived AI index live in two separate local SQLite databases:

~/Library/Application Support/FolderBase/folderbase.sqlite
~/Library/Application Support/FolderBase/folderbase-index.sqlite

  • The watched folders are never modified: FolderBase writes no hidden files inside your directories.
  • To start fresh, just delete that file: it will be recreated empty on the next launch.
  • Metadata is tied to the file's identity, not its path: that's why it "follows" a file when you rename or move it on the same volume.

FAQ

My metadata disappeared after I moved a file.

It should follow automatically on the same volume. If you move across different volumes (e.g. an external drive) the filesystem identity changes and metadata may not migrate: use Maintenance → Check and repair.

I want to reset everything.

Quit the app and delete ~/Library/Application Support/FolderBase/folderbase.sqlite. Your folders and files stay intact.

Does FolderBase modify my files?

No. It only reads name, type, date and size. Everything you add lives in the separate database.

Performance and responsiveness

Table index construction, filtering and sorting run in the background and are cancelled when you change folder or query. Recursive document collection for folder chat also runs away from the UI thread.
